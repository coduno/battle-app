README

Coduno Battle is a game-based programming challenge. Your task as coder is to control a character in a two-dimensional virtual world. All character types are connected to coding, networks and similar stuff as well. Probably you want to automate all your commands by writing some code yourself. You get to pick one of four character types when starting the challenge.

Controlling your character you can issue commands to move your character, attack others and cast spells.

Every character has health points (we like to visualize it as free memory) of 100 bytes/health points.

Interfacing with the game server

Commands are sent to the game server via HTTP. We have prepared a Java library to help you bootstrap on the JVM (Java, Groovy, Scala, ...). It's published on maven as 'uno.cod.battle:java-client:1.0'.

To directly interface, use the following endpoints on battle.cod.uno:8080:

GET /me
Will return details about your character, like health points.
GET /map
Will return detauls about the surroudings of your character. Use this to navigate.
POST /join
This will be the first call you want to make. You'll have to specifiy your character type like { "type": "JAVA_GUY" }
POST /move
Your body should look like { "direction": "UP" }
POST /attack
Your body should look like { "direction": "UP" }
POST /spell
Your request must include a spell specification like { "spell": 2, "direction": "UP", "extra": "" }

If all goes well, you'll get an HTTP status OK. If your command cannot be fulfilled for some reason (e.g. you are on cooldown) you'll receive an HTTP status Bad Request with an error similar to {"battleError": {"code": "", "message": "invalid target" }, "type": "InfoError" }.

To authenticate with the server, you'll need to set the HTTP Authorization header to a value of Token mytoken where mytoken is to be substituted of course. You'll nee the leading "Token ", however.
